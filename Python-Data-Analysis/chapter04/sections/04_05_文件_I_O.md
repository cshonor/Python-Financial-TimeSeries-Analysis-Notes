## 核心一句话

NumPy 用 **`.npy`（单数组）/ `.npz`（多数组）** 二进制格式存数组，**速度快、体积小、保留 dtype/shape**；**日常表格数据优先 `pd.read_csv`**。

---

## 一、书本原文核心知识点提炼

### 4.5.1 `np.save` / `np.load`（单数组，`.npy`）

#### 保存：`np.save(file, arr)`

| 项 | 说明 |
|----|------|
| 格式 | 二进制 **`.npy`** |
| 数量 | **只存一个数组** |
| 文件名 | 未写后缀时 **自动补 `.npy`** |
| 优点 | 比文本快很多，完整保留 dtype、shape |

```python
import numpy as np

arr = np.array([[1, 2, 3], [4, 5, 6]])
np.save("my_arr", arr)      # → my_arr.npy
```

#### 读取：`np.load(file)`

```python
arr2 = np.load("my_arr.npy")
# [[1 2 3]
#  [4 5 6]]
```

---

### 4.5.2 `np.savez`（多数组，`.npz`，未压缩）

#### 保存多个数组到一个文件

| 项 | 说明 |
|----|------|
| 格式 | **未压缩 `.npz`**（实为 zip 包） |
| 用法 | 位置参数或 **关键字命名（推荐）** |

```python
a = np.arange(5)
b = np.array([10, 20])

np.savez("multi_arr.npz", a=a, b=b)
# 或 np.savez("multi_arr.npz", a, b)  # 默认名 arr_0, arr_1
```

#### 读取 `.npz` → 字典式访问

```python
data = np.load("multi_arr.npz")

print(data["a"])    # [0 1 2 3 4]
print(data["b"])    # [10 20]

data.close()        # 好习惯；或用 with np.load(...) as data:
```

---

### 4.5.3 `np.savez_compressed`（压缩版 `.npz`）

- 用法同 `savez`，**体积更小、写入略慢**
- 适合大数据归档、分享

```python
np.savez_compressed("multi_compressed.npz", a=a, b=b)
```

---

### 4.5.4 实务重点：NumPy vs pandas（必背）

| | NumPy（本章） | pandas（实务主力） |
|---|---------------|-------------------|
| **格式** | `.npy` / `.npz` | `.csv`、Excel、Parquet 等 |
| **典型 API** | `np.save` / `np.load` | **`pd.read_csv()`** |
| **适合** | 中间结果、模型参数、纯数值 ndarray | **表格**：表头、字符串、日期、多列异构 |
| **速度/体积** | 二进制，快且小 | CSV 文本慢、占空间大 |

> **数组用 `np.save` / `np.load`；表格用 `pd.read_csv`！**

第 6 章系统讲 pandas I/O；本章只需建立 **ndarray 二进制 vs 表格文本** 的分工。

---

### 4.5.5 易错点（常问）

1. **`.npy` 只能存一个数组；`.npz` 可存多个**
2. **`np.load` 读 `.npz` 返回类字典对象**，必须用 **`data["key"]`** 取数组
3. **默认 `allow_pickle=False`**：含 Python 对象数组会报错；确有需要才 `allow_pickle=True`（**安全风险**，勿加载不可信文件）
4. **CSV 等文本**：慢、占空间大；**npy/npz 二进制**：快、小
5. **`np.save` 覆盖同名文件**无提示，生产环境注意路径与版本

## 二、一页式速查清单

### 命令对照

| 操作 | API | 后缀 | 数组个数 | 压缩 |
|------|-----|------|----------|------|
| 存单数组 | `np.save("x", arr)` | `.npy` | 1 | 无 |
| 读单数组 | `np.load("x.npy")` | `.npy` | 1 | — |
| 存多数组 | `np.savez("x.npz", a=a, b=b)` | `.npz` | ≥1 | 无 |
| 存多数组（压） | `np.savez_compressed(...)` | `.npz` | ≥1 | ✅ |
| 读多数组 | `np.load("x.npz")["a"]` | `.npz` | — | — |
| 读表格 | **`pd.read_csv("x.csv")`** | `.csv` | — | — |

### 记忆口诀

> **单数组 npy，多数组 npz；save 存 load 读，表格 read_csv。  
> npz 像字典要 key，pickle 默认关，二进制快又省。**

## 三、演示

```python
import numpy as np

# 模拟回测中间结果
weights = np.array([0.2, 0.3, 0.5])
nav = np.cumprod(1 + np.array([0.01, -0.005, 0.02]))

np.savez_compressed(
    "backtest_cache.npz",
    weights=weights,
    nav=nav,
)

with np.load("backtest_cache.npz") as data:
    w = data["weights"]
    n = data["nav"]
```

## 四、【量化专属改造】

1. **因子矩阵缓存**：大规模 `(n_date, n_stock)` ndarray → `savez_compressed`，比反复读 CSV 快。
2. **训练/验证划分**：`np.savez("split.npz", X_train=X_train, y_train=y_train, ...)` 一次打包。
3. **勿用 npy 存行情表**：带 `date`、`code`、字符串行业 → **pandas + parquet/csv**（第 6 章）。
4. **时序预测册**：如 `train_val_test.npz` 即多数组 npz 典型用法。
5. **加载不可信 npz**：禁止随意 `allow_pickle=True`（可执行恶意 pickle）。

---

## 五、与 statsmodels / pandas 的衔接

- 回归前：`df = pd.read_csv(...)` → `X = df[["x1","x2"]].to_numpy()`；**持久化 X/y 用 npz**，不是 np.save 两个独立 npy（savez 更方便）。
- 模型系数向量 `params`：`np.save("ols_params.npy", fit.params.values)` 快速快照。
- 全书流程：**CSV 读入（pandas）→ 清洗 → `.values`/`.to_numpy()` → 数值计算（NumPy）→ 中间结果 npz 缓存**。
