# 20 性能与新手避坑指南（写得快、跑得快、结果还对）

> 目标：帮你避开“NumPy 新手最常见、最隐蔽”的坑：慢、错、内存爆、结果莫名其妙。

---

## 1. 什么时候用 NumPy（什么时候不要用）

- **用 NumPy**：
  - 大量数值计算、向量化、矩阵运算
  - 回测核心路径（收益、持仓、指标）需要快
- **不要强行用 NumPy**：
  - 字符串处理、复杂分组逻辑（交给 Pandas/Polars/SQL）
  - 逻辑过于分支化、很难向量化的流程（先写清楚，再优化）

---

## 2. 向量化优先：少写 Python for 循环

典型替代：

- 条件判断：`np.where`
- 过滤：布尔掩码 `x[mask]`
- 累计：`np.cumsum / np.cumprod`
- 滚动均线：可用卷积 `np.convolve`（边界要自己理解）

---

## 3. dtype 选错 = 性能掉一截 + 结果可能错

- 尽量保证数值数组是 `float32/float64/int64`，而不是 `object`
- 从 CSV/列表读入后如果变成 `object`，很多操作会变慢

检查：

- `arr.dtype`

---

## 4. 视图 vs 拷贝（最容易“改了不生效/莫名改到原数据”）

你需要知道两件事：

- **切片**（`a[1:10]`）很多时候是 **view**（共享内存）
- **高级索引**（`a[[1,3,5]]`、`a[mask]`）通常是 **copy**

建议：

- 需要“独立副本”时显式 `.copy()`
- 需要“原地修改”时确认你拿到的是 view

---

## 5. 广播很好用，但 shape 要先想清楚

常见错误：

- `(n,)` 和 `(n,1)` 在矩阵运算里不是一回事

建议：

- 写关键矩阵运算前先 `print(arr.shape)`
- 需要列向量时：`x.reshape(-1, 1)`

---

## 6. 尽量减少临时数组（内存与速度的隐藏杀手）

链式写法可能会创建多个临时数组：

- `y = (a - b) / (c + d)`

优化思路（先正确，再考虑）：

- 用就地运算（`out=`）或分步复用数组
- 但不要为了快把代码写到不可读

---

## 7. 排序/选 TopK：别总是全排序

如果你只要 TopK（量化选股常见）：

- 全排序：`np.argsort(x)[-k:]`（O(n log n)）
- 更快：`np.argpartition(x, -k)[-k:]`（近似 O(n)）

然后再对 TopK 内部做一次排序即可。

---

## 8. 量化实践建议：先“正确”，再“快”

最稳妥的节奏：

1. 先用清晰写法跑通（含对齐、缺失值、shift）
2. 加自检（是否未来函数、是否 NaN 爆炸、shape 是否对）
3. 再把热点路径改成向量化/减少临时数组
