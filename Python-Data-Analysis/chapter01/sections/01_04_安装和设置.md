# 1.4 安装和设置

> 所属：[第 1 章](../chapter01_getting_started.md) · 《利用 Python 进行数据分析》

**核心要点**：**不存在唯一正确的 Python 环境**，按用途（数据分析 / 科研 / 量化 / Web）选择 **conda / venv / Docker** 等即可；数据科学优先 **Miniconda**。

## 一、书本原文核心知识点提炼

### 1.4 安装和设置

原书强调：环境是手段不是信仰；关键是 **隔离、可复现、与项目匹配**。

#### 1. 环境选择总览

| 方案 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| **Conda（Miniconda）** | 跨语言包管理、二进制依赖、虚拟环境 | 比 venv 略重 | **数据分析首选**（本书推荐） |
| **venv** | 内置、轻量、官方标准 | 不管理 C 扩展等非 Python 依赖 | Web、脚本、纯 Python |
| **Docker** | 完全隔离、跨平台一致 | 学习成本、镜像体积 | 团队、CI/CD、教学分享 |
| **pip + 系统 Python** | 最简单 | 易污染系统、依赖弱 | 仅临时小脚本 |

> 推荐：**Miniconda（轻量）**，不必装臃肿的 Anaconda 全家桶。

#### 2. Miniconda 安装要点

- 下载：[Miniconda 官方文档](https://docs.anaconda.com/miniconda/)
- Windows：64-bit 安装包，安装时勾选 **Add to PATH**（或安装后用 Anaconda Prompt）
- macOS / Linux：`bash Miniconda3-xxx.sh`，同意写入 `.bashrc` / `.zshrc`
- 验证：`conda --version`、`python --version`（建议 **Python 3.10+ / 3.11+**）

#### 3. 虚拟环境（必做：一项目一环境）

避免升级一个包导致全局崩溃；量化/笔记仓库建议 **每本书或每个大项目** 独立环境。

#### 4. 本书核心栈

NumPy、pandas、Matplotlib、Seaborn、scikit-learn、Jupyter；推断向后续加 **statsmodels**（第 12 章）。

#### 5. 国内镜像

Conda / pip 换清华等镜像可显著加速（见下方速查）。

#### 6. IDE

- 入门交互：**Jupyter Notebook / Lab**
- 日常开发：**VS Code**（选对解释器）、PyCharm

## 二、关键语法速查表（命令行）

### Conda 环境（推荐）

```bash
# 创建
conda create -n pandas-book python=3.11

# 激活 / 退出
conda activate pandas-book
conda deactivate

# 列表
conda env list

# 导出 / 复现（团队必备）
conda env export > environment.yml
conda env create -f environment.yml
```

### venv 环境（备用）

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

pip install numpy pandas matplotlib seaborn scikit-learn jupyter
```

### 安装本书核心库

```bash
# Conda（推荐，自动处理二进制依赖）
conda install numpy pandas matplotlib seaborn scikit-learn jupyter

# 或 pip
pip install numpy pandas matplotlib seaborn scikit-learn jupyter

# 本仓库后续章节常用（可选一次装齐）
pip install scipy statsmodels linearmodels wooldridge
```

### Jupyter

```bash
jupyter notebook
# 或
jupyter lab
```

### 国内换源（可选）

```bash
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --set show_channel_urls yes

pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### Docker（可选）

```dockerfile
FROM continuumio/miniconda3
WORKDIR /app
COPY environment.yml .
RUN conda env create -f environment.yml
SHELL ["conda", "run", "-n", "pandas-book", "/bin/bash", "-c"]
COPY . .
CMD ["conda", "run", "-n", "pandas-book", "jupyter", "notebook", "--ip=0.0.0.0", "--allow-root"]
```

```bash
docker build -t pandas-env .
docker run -p 8888:8888 pandas-env
```

## 四、【量化专属改造】

1. **量化推荐栈**：`Miniconda + python=3.11` + `numpy pandas scipy statsmodels linearmodels jupyter`；深度学习按需加 PyTorch（见时序预测册）。
2. **勿用系统 Python 跑回测**：固定 `environment.yml` 或 `requirements.txt` 提交到 Git，保证 **复现**。
3. **多书 mono-repo**：可建一个 `py311-quant` 环境覆盖五本书依赖，或按书拆分（Data-Analysis / Econometrics / Hilpisch 各一）。
4. **Windows 注意**：路径含中文时偶发编码问题；项目路径尽量英文，数据可中文文件名但需测试 `pd.read_csv` 编码。
