## 一、书本原文核心知识点提炼

### 12.1 何时使用深度学习进行时间序列预测

- **规模**：通常 **>10,000** 点较理想；约 **8,000** 也可作为起点（非绝对硬门槛）。
- **统计模型失效场景**  
  - **SARIMAX 拟合过慢**：大样本 + 大量候选模型时网格不可接受。  
  - **多重季节**：如小时数据同时有**日季节 + 年季节**，SARIMA 难以原生表达，残差常仍相关。  
- **DL 优势**：融合多周期季节、捕捉**非线性**、大样本下训练仍较快。

### 12.2 探索不同类型的深度学习模型

| 类型 | 输出 | 示例 |
|------|------|------|
| **单步 (Single-step)** | 标量，未来 **1** 步、**1** 个目标 | 下一小时流量 |
| **多步 (Multi-step)** | 序列，**1** 目标 × **H** 步 | 未来 24 小时流量 → \(24\times1\) |
| **多输出 (Multi-output)** | 多目标，可单步或多步 | 24 步 ×（流量+温度）→ \(24\times2\) |

输入序列长度可任意；差异在**输出张量形状**。

### 12.3 准备应用深度学习进行预测

**12.3.1 数据探索**

- 数据：I-94 **西行**小时交通；**17,000+** 行。  
- 目标：预测 `traffic_volume`（单步/多步）；亦可同时预测流量与 `temp`（多输出）。  
- **流量**：强**日季节**（清晨/深夜低，周末更低）。  
- **温度**：**年季节**（冬低夏高）+ **日季节**（夜间低、白天高）。

**12.3.2 特征工程与拆分（关键）**

- **删列**：`describe()` 见 `rain_1h` 几乎全 0、`snow_1h` 全 0 → 删除。  
- **周期时间编码（重点）**  
  - 勿仅用单调递增时间戳（会**丢失周期**）。  
  - 用当日秒数映射到弧度：`sin`、`cos` 将时间折成**表盘圆**，保留 24h 循环（如 5:00 与 17:00 可区分）。  
  - 编码后删除原 `date_time` 字符串列。  
- **拆分**：**70% 训练 / 20% 验证 / 10% 测试**，**禁止打乱**时间顺序。  
- **缩放**：`MinMaxScaler` **仅在训练集 `fit`**，再 `transform` 验证/测试；DL **不要求**正态分布，只缩放到合适数值范围。

- **练习**：北京奥体中心空气污染数据 — 删无用列 → 识别日季节 → sin/cos 编码 → 70:20:10 → MinMaxScaler → **保存**供后续章节。

## 二、关键语法速查表

| 场景 | 代码要点 |
|------|----------|
| 日周期编码 | `rad = 2π * seconds_in_day / 86400`；`sin`, `cos` |
| 时序拆分 | `n_train = int(n * 0.7)` 等，按索引切片 |
| 缩放 | `scaler.fit(train)`；`transform(val/test)` |
| 模型形态 | 输出 shape：`(1,)` / `(H,)` / `(H, K)` |

## 四、【量化专属改造】

| 场景 | 做法 |
|------|------|
| 分钟/ tick 量 | >10k 点后考虑 DL；加交易时段 sin/cos |
| 多标的 | 多输出头或 per-symbol 模型 |
| 防泄漏 |  scaler、特征统计量仅 fit 训练段 |
| 仍可用统计 | 小样本、单季节、需可解释 → SARIMA 仍优先 |
