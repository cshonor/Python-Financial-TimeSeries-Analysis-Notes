## 底层原理

虽然抽象逻辑上是一个面阵结构，但在物理内存中都是一维线性排列。陈列方式直接决定 CPU 缓存命中（Cache Hit）几率。

| 布局 | 内存排布 |
|------|----------|
| **C 连续（C-order）** | 按**行**连续排布（默认） |
| **Fortran 连续（F-order）** | 按**列**连续排布 |

```python
c_arr = np.array([[1, 2, 3], [4, 5, 6]], order='C')
f_arr = np.array([[1, 2, 3], [4, 5, 6]], order='F')
c_arr.flags['C_CONTIGUOUS']  # True
```

## 性能实操结论

- C 连续数组在执行**跨行、按行扫描**的聚合时效率更高
- 按列连续排布时，**纵向切取**才能获得更好局部性
