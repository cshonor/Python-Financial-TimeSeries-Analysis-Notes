# 4.2.6 速度对比

> 所属：[第 4 章](../chapter04_numpy_numerical_computation.md) · 《Python金融大数据分析（第2版）》

---

## 核心论据

构建 \(5000 \times 5000\) 的巨型随机数网格，对比纯 Python 循环与 NumPy 聚合的性能差异。

| 方式 | 典型耗时 |
|------|----------|
| 双重 `for` + `sum` 穷举求和 | 约 17 秒及以上 |
| `ndarray.sum()` | 不到 30 毫秒 |

## 关键结论

NumPy 操作不但将**代码排布密度**进行了降维打击，底层以 C 为根基的执行层面对比纯 Python 存在**数十倍甚至上百倍**的速度优势。

运行本章脚本中的 `demo_speed_comparison()` 可在本机复现量级差异（绝对数值因硬件而异）。
