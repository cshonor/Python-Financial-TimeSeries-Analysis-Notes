# 4.2.4 改变组成与大小

> 所属：[第 4 章](../chapter04_numpy_numerical_computation.md) · 《金融数据分析及其 Python 应用》

---

## 核心概念

大量此类操作返回的仅为原数据的**新视角映射（View）**，从而避免在大规模金融数据集上重新划定内存的开销。

## 结构重塑与合并

| API | 行为 |
|-----|------|
| `reshape()` | 在总元素数量不变前提下改变维度（如 12 元素 → 3×4） |
| `resize()` | 强制改写规模；新体积超载时循环复制源数据填充 |
| `np.hstack()` | 横向拼接（增加列） |
| `np.vstack()` | 纵向拼接（增加行） |

## 多维坍缩为一维

| API | 策略 |
|-----|------|
| `flatten()` | 压扁成一条线，**生成独立内存副本** |
| `ravel()` | 与 `flatten` 效果一致，但优先返回**视图**，性能更高 |
